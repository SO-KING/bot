# HopX Telegram Bot

بوت تيليجرام يعمل كجسر بينك وبين منصة HopX:
- يسجّل مفتاح API الخاص بك (مشفّر بـ Fernet)
- ينشئ لك سيرفر (sandbox) على HopX
- يمنحك تحكماً كاملاً عبر تيليجرام:
  - **ترمنال تفاعلي حقيقي** (stateful WebSocket — `cd` و `export` تبقى فعّالة بين الأوامر)
  - **إدارة ملفات** ( تصفّح، رفع، تحميل، حذف، إنشاء مجلدات)
  - **إدارة العمليات** (عرض عمليات النظام أو عمليات الخلفية، إيقاف أي عملية)
  - **حالة السيرفر** (وقت انتهاء، تمديد المهلة، إيقاف/استئناف، إنهاء)
- كل رسالة يرسلها البوت يظهر تحتها شريط حالة مختصر فيه معرّف السيرفر، حالته، والوقت المتبقي.
- كل العمليات تتم على سيرفر المستخدم نفسه — لا يمكن لأي مستخدم تنفيذ أوامر على سيرفر آخر أو على جهاز البوت نفسه.

## المتطلبات

- Python 3.11 أو أحدث
- مفت HopX API من https://console.hopx.dev
- توكن بوت تيليجرام من @BotFather

## التثبيت

```powershell
cd C:\hopx-bot
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
```

### توليد مفتاح Fernet

```powershell
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```
انسخ الناتج والصقه في `FERNET_KEY` داخل `.env`.

### ضبط `.env`

```
TELEGRAM_BOT_TOKEN=123456:ABCDEF...
FERNET_KEY=...
HOPX_DEFAULT_TEMPLATE=code-interpreter
HOPX_DEFAULT_TIMEOUT=3600
```

### إعدادات BotFather الموصى بها

- **Privacy mode**: مفعّل — حتى لا يستهلك البوت رسائل لا تخصه ويبقى مفتاح المستخدم خاصاً.
- اختياري: قائمة الأوامر `/start /help /cancel /reset`.

## الاستخدام

1. ابدأ البوت بـ `/start` — سيطلب مفتاح HopX.
2. أرسل المفتاح (يبدأ بـ `hopx_live_`) — البوت يتحقق منه، يشفره، ويحفظه في قاعدة بيانات محلية.
3. اضغط **🚀 إنشاء سيرفر جديد** — ينشئ سيرفر على HopX خلال ثوانٍ.
4. بعد الإنشاء يفتح لك القائمة الرئيسية مع:
   - 🖥️ الترمنال — يفتح جلسة WebSocket تفاعلية. اكتب أي أوامر هنا في ردّ رسالة عادية. اضغط ⌨️ وضع الإدخال لإرسال بدون Enter (للبرامج التي تطلب input). أزرار Ctrl+C، Clear، إغلاق متوفرة.
   - 📁 إدارة الملفات — تصفّح `/workspace`، ارفع ملف (أرسله كـ Document)، حمّل ملف (يصلك كـ Document)، احذف (مع تأكيد)، أنشئ مجلد.
   - ⚙️ العمليات — بدّل بين عمليات النظام وعمليات الخلفية، اضغط 🛑 لإيقاف أي عملية.
   - 📊 حالة السيرفر — اطلع على معلومات السيرفر ومدّد المهلة (+1h / +3h) أو أوقف/استأنف أو أنهِ السيرفر.
5. تحت كل رسالة من البوت يظهر شريط حالة مع زر "🏠 الرئيسية" للرجوع السريع.

## الأمان

- مفتاح HopX API يُشفّر بـ Fernet قبل تخزينه في `hopx_bot.db`. يُفكّ التشفير فقط في الذاكرة عند الاستدعاء.
- كل مستخدم معزول بـ `user_id` — لا يمكن مستخدم الوصول لسيرفر مستخدم آخر.
- البوت يتعامل فقط مع API HopX؛ فشل أي أمر على HopX لا يؤثر على جهاز البوت نفسه أو على بوتات/مستخدمين آخرين.

## حدود HopX المهمة

- مسارات الملفات مسموحة فقط: `/workspace` و `/tmp`.
- مهلة التنفيذ المتزامن الأقصى 300 ثانية. للأطول استخدم الترمنال المتفاعل أو تشغيل في الخلفية.
- السيرفر ينتهي تلقائياً عند انتهاء timeout — مدّده من قبل أن ينتهي أو استأنفه مع `resume`.

## هيكل المشروع

## ملاحظة

هذا البوت بأداة إدارية وليس للإنتاج متعدد المستخدمين بدون تعديلات (نظير auth وتقييد user_ids). راجع `bot.py` وأضف whitelist للمستخدمين المصرّح لهم إذا رغبت.
